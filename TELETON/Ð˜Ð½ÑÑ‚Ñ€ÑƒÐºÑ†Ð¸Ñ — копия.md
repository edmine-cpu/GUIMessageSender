# Teleton — инструкция по интерфейсу

Эта инструкция объясняет, что делает каждая вкладка и как запустить основные сценарии.  
Внутри программы есть встроенная справка: кнопка **❓ Справка (F1)** в левом меню.

---

## Быстрый старт (5 шагов)

1. Открой **Аккаунты/Прокси** → добавь аккаунт (TData или `.session`).
2. Проверь, что аккаунт **подключается** и статус `active`.
3. Открой **Шаблоны чатов/каналов** → сохрани список чатов/каналов (по одному на строку).
4. Открой нужный раздел (Парсинг / Рассылка / Комментинг) → выбери аккаунт и шаблон.
5. Включи **Dry Run** (если доступно) для проверки, затем запускай в Live.

---

## Общие понятия

### Аккаунт и статус

В базе у каждого аккаунта есть статус:
- `active` — работает.
- `needs_reauth` — сессия устарела/отозвана, нужен переимпорт.
- `network_issue` — проблемы сети/прокси (временный статус).
- `banned` — аккаунт забанен Telegram.

Если какая-то функция не запускается — первым делом смотри статус аккаунта и лог.

### Proxy

Прокси можно назначать на аккаунт.  
Рекомендуется использовать прокси для аккаунтов и для AI-запросов (если используешь AI-функции).

### Dry Run

Dry Run — режим проверки без реальной отправки.  
Если включён, модуль пишет в лог, что **он бы сделал**, но **не отправляет сообщения**.

## Разделы интерфейса

### 1) Аккаунты/Прокси

Зачем: добавить аккаунты Telegram и назначить прокси.

Основные действия:
- Импорт TData (частый сценарий при покупке аккаунтов).
- Импорт `.session`.
- Назначение прокси аккаунту.
- Проверка статуса и ошибок подключения.

Если не подключается:
- `needs_reauth` → переимпортируй TData/сессию.
- `network_issue` → проверь прокси/интернет.

---

### 2) Шаблоны чатов/каналов

Зачем: хранить списки для повторного использования (чаты/каналы).

Как пользоваться:
- Создай шаблон.
- Вставь ссылки/юзернеймы по одному на строку.
- В других разделах нажимай «Шаблон» и выбирай нужный список.

---

### 3) Парсинг

Зачем: собрать пользователей в базу для дальнейших действий (аудитории / DM-рассылка).

Режимы:
- Обычный парсинг участников.
- Aggressive (для больших групп).
- Комментаторы каналов (если комментарии включены).
- Смарт-парсинг: `keywords` или `ai` (по смыслу через AI).

Результат: пользователи сохраняются в базу, затем можно собрать аудиторию.

---

### 4) Аудитории

Зачем: сохранить фильтрованный список пользователей и использовать его в DM-рассылке.

Как пользоваться:
- Создай аудиторию из базы parsed_users.
- Выбери аудиторию в DM-разделе и запусти рассылку.

---

### 5) Рассылка

Внутри несколько вкладок:
- **Упоминания** — упоминания пользователей в группах.
- **Рассылка** — берёт цели из очереди задач (тип `broadcast`) и отправляет по группам.
- **Циклическая** — ходит по списку целей по кругу и продолжает после перезапуска.
- **Задачи** — очередь целей для вкладки «Рассылка».

Уникализация текста:
- **Оригинал** — без изменений.
- **Спинтакс** — `{вариант1|вариант2}`.
- **Омоглифы** — подмена символов для лёгкой уникализации.
- **AI** — перефразирование через OpenAI/Groq.

#### Циклическая рассылка (важное)

Перед стартом выводится «план запуска»:
- какие аккаунты участвуют и какие прокси у них,
- какие чаты и сколько целей,
- какой источник текста,
- задержка min/max и пауза круга,
- Dry Run,
- текущая позиция цикла (сохранённое состояние),
- что будет происходить после запуска.

Состояние цикла сохраняется, поэтому после перезапуска продолжает с последней позиции.

---

### 6) Комментинг

Два режима:
- **Старые посты** — комментирует последние N постов в каждом канале.
- **Новые посты** — слушатель новых постов и авто-комментирование.

AI-комментарии:
- Если включить **AI-комментарии**, комментарий генерируется по тексту поста.
- Можно настроить промпт, тон и длину, провайдера (openai/groq).
- Есть **Dry Run**: показывает комментарий, но не отправляет.

---

### 7) Автоответчик

Зачем: автоматически отвечать на входящие личные сообщения.

Как пользоваться:
- Выбери аккаунт.
- Вставь шаблон ответа.
- Запусти. Остановить — завершает listener.

Если не запускается — в логе будет причина (подключение/статус аккаунта).

---

### 8) Аккаунт

Зачем: управление профилем и очистка аккаунта.

Профиль:
- Имя/Фамилия/Bio/Аватар (в т.ч. случайный из папки).
- Если Telegram запретил изменение (например аккаунт frozen) — будет понятная причина в логе.

Очистка:
- Выход из групп/каналов, удаление диалогов и т.п.  
Это необратимо — делай аккуратно.

---

### 9) Объявления

Отдельный модуль планировщика публикаций:
- Объявления (тексты/медиа),
- Группы (куда публиковать, интервалы и часы),
- Планировщик (старт/стоп),
- История публикаций.

Есть AI-генерация текста объявлений + выбор провайдера.

---

### 10) Логи/Статистика

Зачем: смотреть, что реально произошло (успехи/ошибки/flood/banned).

Если что-то “не работает”, почти всегда причина уже есть в логе.

---

### 11) Настройки

Здесь задаются:
- задержки min/max для разных режимов,
- AI ключи и прокси (OpenAI/Groq),
- таймауты импорта TData и другие параметры.

Важно: если AI используется без прокси, может светиться реальный IP.
